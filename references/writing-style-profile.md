# 文风特征档案

> 基于 12 篇代表性样本交叉分析，提炼商务方案写作的稳定风格特征。
> 本档案服务于 proposal-logic-framework.md，为方案生成提供"怎么写"的风格指导。

---

## 一、风格核心概要

**一句话定义：** 体系化思维驱动的实操型方案文风——用严密的层级结构把复杂问题拆解为可执行的模块，语言克制精准，重逻辑完整性和可操作性，轻修辞和情感渲染。

**风格稳定性：** 高。样本跨越多种方案类型，核心风格特征高度一致。

---

## 二、宏观风格特征（跨文本恒定）

### 2.1 思维模式

- **系统分解：** 任何复杂问题都被拆解为模块，模块之间有明确的逻辑关系和先后顺序。不存在"一段话说完一件大事"的情况，一定是分层、分项、分步展开。
- **过程导向：** 强调"怎么做"的完整路径，而非仅描述"做什么"。每个方案都有清晰的阶段划分和步骤编排。
- **闭环驱动：** 每个动作都有对应的检验机制——责任人对应检查人，执行对应验收，结果对应运用。不留开放性的"待定"。
- **穷举倾向：** 倾向于列举所有可能的情况、所有涉及的角色、所有需要的工具，追求覆盖的完整性。

### 2.2 价值取向

- **实效导向：** 方案的核心追求是"能落地、能执行、能见效"，不追求理论深度或概念创新。
- **过程规范：** 强调按规则办事，过程透明可追溯，决策有据可依。
- **数据说话：** 偏好用量化指标定义目标和衡量结果，能量化的绝不定性描述。
- **持续改进：** PDCA思维贯穿始终，方案不是一次性的，而是"执行→评估→优化→再执行"的循环。

### 2.3 表达意图

- **双重读者：** 方案同时面向决策者（审批）和执行者（操作指南），因此既有宏观框架又有微观细节。
- **行动导向：** 每一段文字都指向一个具体的行动或决策，不做纯粹的分析或讨论。
- **确定而不居高临下：** 语气确定、专业，但不说教。用"通过XX实现XX"而非"应该XX"。

### 2.4 风格定位

- 正式商务文书，介于政府公文和管理咨询报告之间
- 有企业正式文书的底色（编号体系、审批链、过程留痕），融入现代管理方法论
- 不是学术论文风格，不引用文献；不是PPT风格，不用碎片化表达

## 三、微观风格特征（跨文本恒定）

### 3.1 句式特征

**主力句式（占比约70%）：**

① **目的句：** "为{目标}，通过{手段}，实现/达到{预期效果}"
```
句式模板：为解决{问题}/提升{指标}/完善{机制}，通过{方法}，实现{可衡量的成果}，特制定本方案。
```

② **并列枚举句：** "1、{要点一}；2、{要点二}；3、{要点三}"
```
句式模板：
1、{动词}{对象}，{补充说明预期效果}；
2、{动词}{对象}，{补充说明预期效果}；
3、通过{手段}，为{受益方}{创造/提供}{价值}。
```

③ **条件-动作句：** "根据/基于{条件}，结合{情境}，{动作}"
```
句式模板：根据{上级要求/行业标准/实际情况}，结合{本单位/本项目的具体条件}，制定本方案。
```

④ **定义句：** "{概念}是指{解释}，包括{列举}"
```
句式模板：{术语}是指{一句话定义}。包括但不限于：1、{类别一}；2、{类别二}；3、{类别三}。
```

**辅助句式（占比约30%）：**

⑤ **原则阐释句：** "{原则名}原则：{一句话解释}"
```
句式模板：{四字/六字原则名}原则：{用一句话说明该原则对方案设计的约束或指导}。
```

⑥ **职责分配句：** "{角色}负责{职责内容}"
```
句式模板：{角色名称}负责{动宾短语描述的具体职责}，{补充说明权限边界或协作关系}。
```

### 3.2 用词特征

**按功能分类的高频动词：**

| 功能 | 动词 | 使用场景 |
|------|------|---------|
| 启动 | 拟定、制定、编制、设计、梳理 | 方案/标准/流程的创建 |
| 执行 | 推进、开展、实施、落实、执行 | 行动项的描述 |
| 产出 | 形成、出具、提报、汇总、固化 | 交付物的描述 |
| 审批 | 审核、审批、报审、审定 | 决策流程的描述 |
| 沟通 | 宣贯、沟通、公示、反馈 | 信息传递的描述 |
| 提炼 | 萃取、沉淀、提炼、总结、归纳 | 经验/知识的积累 |

**按功能分类的高频名词模式：**

| 功能 | 名词模式 | 说明 |
|------|---------|------|
| 组织 | {项目/专项}组、{领域}委员会、{职能}小组 | 根据方案性质命名 |
| 文档 | {方案名}实施方案、推进计划、{领域}标准 | 根据内容命名 |
| 机制 | {功能}机制（如预警机制、反馈机制、升级机制） | 根据解决的问题命名 |
| 标准 | {对象}{维度}标准（如验收标准、操作标准） | 根据评估对象命名 |
| 证据 | 见证性资料、输出物、交付物 | 过程留痕的统称 |

**连接词偏好：**
- 因果：鉴于、基于、据此、因此、特制定
- 并列：同时、并、以及
- 递进：进一步、从而、以达到
- 限定：不少于、不超过、不低于、占比XX%

**回避的表达：**
- 口语化（"搞一个""弄一下"）
- 模糊限定（"大概""差不多""可能"）
- 情感渲染（"非常重要""极其关键""迫在眉睫"）
- 第一人称（"我认为""我们觉得"）

### 3.3 段落与结构特征

- **段落短小：** 每个编号项通常1-3句话，极少超过5句。长内容通过增加编号层级拆分，而非写长段落。
- **表格密集：** 凡涉及多维度对比、时间计划、标准分级、角色分工的内容，一律用表格呈现。
- **附件承载细节：** 正文讲框架和逻辑，具体的表单模板、操作细则、工具流程放入附件。正文引用附件但不展开。
- **首尾闭环：** 开头的"目的"和结尾的"结果运用"形成呼应。

### 3.4 情感基调

- **整体克制：** 不煽情、不夸张、不渲染。用事实和数据代替情感表达。
- **问题描述时：** 客观陈述，偶尔用"缺乏""不足""亟待"表达改善需要，但不制造焦虑。
- **方案描述时：** 自信、确定，用"通过XX实现XX"的句式传递可行性。
- **正面语境时：** 偶有少量积极表达，但点到为止，不堆砌形容词。

## 四、适应性特征（随问题类型变化的风格调整）

对应 proposal-logic-framework.md 的九种问题类型：

| 问题类型 | 背景写法 | 量化程度 | 表格侧重 | 节奏感 |
|---------|---------|---------|---------|--------|
| 诊断-改善型 | 直奔问题，数据呈现现状 | 很高（基线→目标对比） | 数据分析表、对策表 | 紧凑、数据密集 |
| 评估-决策型 | 说明为什么需要评估/选择 | 高（维度、权重、公式） | 评估维度表、评分表 | 严谨、层次分明 |
| 行为塑造型 | 展开现状痛点和期望行为 | 中高（标准、阈值） | 标准对照表、反馈流程表 | 条理清晰 |
| 能力建设型 | 盘点现状差距 | 中（目标水平定义） | 差距分析表、培养计划表 | 温和、发展导向 |
| 体系构建型 | 建立全貌，说明为什么需要新体系 | 中 | 架构图、模块分工表、里程碑表 | 系统感、层层递进 |
| 应急响应型 | 简短，直接说明触发条件 | 低（重速度不重精度） | 分级响应表、联络表 | 简洁、指令式 |
| 探索创新型 | 说明机会或不确定性 | 中低（假设性指标） | 实验设计表、决策点表 | 开放、阶段性收敛 |
| 整合对齐型 | 盘点各方差异全貌 | 中 | 差异对比表、过渡计划表 | 稳重、平衡感 |
| 维持运营型 | 简短，说明运营对象和标准 | 高（监控指标、阈值） | 巡检/维护计划表、预警标准表 | 规律、标准化 |

## 五、个性标记模式

### 5.1 理念锚定模式

作者习惯用简洁的短语来锚定方案的核心理念，作为贯穿全文的价值支点。

**模式：** 在背景或原则部分引入一个凝练的理念短语，后续在目的、实施、结果运用等环节反复呼应。

**运用方法：** 根据方案所属领域，提炼或引用该领域公认的理念短语作为锚点。如果客户/组织有自己的核心理念或价值观表述，优先使用。

### 5.2 方法论命名模式

作者习惯将方案采用的方法论浓缩为一个结构化的名称，便于记忆和传播。

**模式：** "{数字}{关键词}"结构，如"五个维度""三步走""四阶段"。

**运用方法：** 将方案的核心方法论提炼为类似的结构化命名，在方案开头引入，后续展开时反复引用。

### 5.3 过程留痕模式

作者对过程证据有强烈偏好，每个行动项都要求产出可验证的实物。

**运用方法：** 推进计划表中，每个工作项必须有对应的输出物列，输出物用《》标注文档名称。

## 六、风格模仿策略

### 6.1 必备元素（缺一则风格失真）

- 严格的层级编号体系（一→（一）→1→①）
- 每个行动项有责任人、检查人、输出物、时间节点
- 原则部分用"{名称}原则：{解释}"格式
- 涉及重要决策时有异议处理通道
- 正文+附件的分离结构

### 6.2 重要元素（应尽量包含）

- 量化目标和衡量标准
- 表格呈现结构化数据
- 领域理念或组织价值观的引用锚点
- 阶段性推进计划表
- 闭环设计（结果运用环节）

### 6.3 风格禁忌（出现则破坏风格）

- 口语化表达
- 模糊限定词（大概、可能、差不多）
- 情感化渲染（非常重要、极其关键、迫在眉睫）
- 第一人称叙述
- 长段落（超过5句不分项的连续文字）
- 无编号的散文式展开

### 6.4 按问题类型适配风格

对应 proposal-logic-framework.md 2.5节的九种问题类型 + 兜底路径：

- **诊断-改善型：** 加重数据密度，突出基线→目标对比，使用该领域的分析工具术语
- **评估-决策型：** 详细展开评估维度和权重设计，决策逻辑要透明可追溯
- **行为塑造型：** 标准定义要清晰具体，反馈机制要及时闭环
- **能力建设型：** 强调差距分析的客观性，建设手段要分类分层匹配
- **体系构建型：** 强调分步推进和阶段验收，架构设计要先全貌后细节
- **应急响应型：** 语言更简洁直接，指令式表达增多，减少论述性内容
- **探索创新型：** 允许更多假设性表述，但每个假设必须有验证方案
- **整合对齐型：** 强调差异的客观呈现和过渡方案的平衡性
- **维持运营型：** 标准化表达为主，重复性工作用表格固化
- **兜底/新类型：** 通过网络搜索获取该领域的成熟方法论和术语体系，风格框架不变，填充领域知识
